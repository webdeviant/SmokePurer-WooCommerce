SmokePurer Sync for WooCommerce — Changelog
============================================

Newest versions are listed at the top. Each release follows the same format:

    Version X.Y.Z

    - Change one.
    - Change two.


Version 1.0.4
-------------

- Images the supplier serves as 404/Not Found are now parked after one attempt and
  are no longer re-downloaded or re-logged on every catalogue run (big reduction in
  wasted requests and log noise). Parked images retry automatically after 30 days.
- Added a "Retry missing images" button (Dashboard > Maintenance) to clear the
  parked list and re-pull on demand (e.g. after the supplier fixes the images); the
  dashboard shows how many images are currently parked.
- Fixed the image-failure log to report the actual number of attempts (a permanent
  404 is tried once, not the retry maximum).


Version 1.0.3
-------------

- Added per-field "keep updating" options (Settings > Ongoing updates) controlling
  what continues to sync on a product AFTER its first import: price, name,
  description, categories, weight, brand, tags and image. New products always import
  in full; stock always syncs via the 5-minute job. Default keeps price/weight/tags
  fresh and freezes presentational content so manual edits are preserved.
- Added an admin warning notice when the stock sync has failed or stopped, so a
  silent stoppage is noticed without opening the dashboard (internal; no external
  services).


Version 1.0.2
-------------

- Added an option to import products without categories (products are left
  unassigned to any category). Toggle "Assign categories" in Settings.
- Refreshed the README documentation (condensed, with branding).


Version 1.0.1
-------------

- Added image download throttling and retry with exponential backoff to avoid
  supplier rate-limiting during large image imports (configurable in Settings).
- Security hardening: feed-supplied image URLs are validated against internal /
  loopback / reserved hosts before download (SSRF protection); the host guard now
  also blocks IPv6-literal and numeric-encoded IP bypasses and resolves hostnames.
- Compatibility: verified clean on PHP 8.3, 8.4 and 8.5 (fixed a PHP 8.4+
  fgetcsv() deprecation).


Version 1.0.0
-------------

- Initial public release.
- Initial WooCommerce integration.
- Catalogue import for simple and variable products with automatic markup pricing.
- Five-minute stock synchronisation from the supplier quantity feed.
- Pre-order handling, sale-price management, images, brands, categories and weights.
- Full-snapshot safety checks: feed validation and circuit-breaker.
- Retirement of discontinued lines from the disabled feed.
- Production-ready release.
